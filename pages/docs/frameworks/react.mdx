1. `/src`

This directory typically contains the source code of your React application.

2. `/public`

This directory holds static assets that you want to be publicly accessible, such as HTML files and images.

3. `/node_modules`

Created when you install project dependencies using a package manager like npm or Yarn. It contains all the libraries and packages your project depends on.

4. `package.json`:

This file lists project metadata and dependencies.
It includes scripts for running, building, and testing your React application.

5. `package-lock.json` or `yarn.lock`:

These files lock the specific versions of dependencies installed in your project, ensuring consistency across different environments.

6. `public/index.html`

The main HTML file where your React application is typically mounted.

7. `src/index.js` or `src/index.tsx`

The entry point of your React application, where you usually import and render the root component (usually `App.js` or `App.jsx`).

8. `src/App.js` or `src/App.jsx`

The main component that serves as the root of your React application.

9. `src/components`

This directory is often used to store reusable React components that are used across different parts of your application.

10. `src/pages` (optional)

In some projects, this directory is used to organize components that represent different pages or views of your application.

11. `src/styles` (optional)

If you're using CSS or a CSS preprocessor like SCSS, this directory is used to store your stylesheets.

12. `src/assets` (optional)

You can use this directory to store assets like images, fonts, or other files used in your application.

13. `src/utils` (optional)

If your project requires utility functions, you can organize them in this directory.

14. `README.md`

A readme file typically provides documentation and information about your project.

Remember that this structure is not set in stone, and you can adapt it to suit your project's specific needs and organizational preferences. Additionally, larger and more complex projects might require more folders and additional configuration files to manage state, routing, or other specific requirements.

## Loops

### Looping through an array

```jsx
let elements = ["one", "two", "three"];

return (
  <ul>
    {elements.map(function (value, index) {
      return <li key={index}>{value}</li>;
    })}
  </ul>
);
```

<Callout type='default'>
  Each list item inside a map loop needs a key attribute with a unique value
  which is generally the index.
</Callout>

### Looping through an array of objects

```jsx
let elements = [
  {
    name: "one",
    value: 1,
  },
  {
    name: "two",
    value: 2,
  },
  {
    name: "three",
    value: 3,
  },

return (
  <ul>

      return (
        <li key={index}>
          The value for {element.name} is {element.value}
        </li>
      );
    })}
  </ul>
);
```

<Callout type='default'>
  Each list item inside a map loop needs a key attribute with a unique value

## Forms

### React forms

```jsx
import React, { useState } from "react";

export default function LoginForm() {
  let [username, setUsername] = useState("");
  let [password, setPassword] = useState("");

  function handleSubmit(event) {
    event.preventDefault();
    alert(`Loging in with ${username} and ${password}`);
  }

  function updateUsername(event) {
    setUsername(event.target.value);
  }

  function updatePassword(event) {
    setPassword(event.target.value);
  }

  return (
    <form onSubmit={handleSubmit}>
      <input type='text' placeholder='Username' onChange={updateUsername} />
      <input type='password' placeholder='Password' onChange={updatePassword} />
      <input type='submit' value='Login' />
    </form>
  );
}
```
